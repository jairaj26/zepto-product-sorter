# 🛒 Zepto Product Sorter (Bookmarklet)

I built this tool because one of the main issues Zepto users face is finding good offers. There is no way to sort items by discount since, unlike Swiggy or BigBasket, Zepto doesn't have a default sorting option.

Zepto also removed the discount percentage tag from their interface, so it was extremely tiresome to guess or calculate how much discount a product actually had.

This is a Javascript bookmarklet that injects a sorting panel into the Zepto website to solve these problems. **Now available for both PC and Mobile!**

---

## 🏗️ Architecture & Execution

This project was built with a strict focus on cost-effective execution. By ruthlessly prioritizing a free, zero-server DOM-parsing architecture, the tool completely avoids expensive backend infrastructure, proxy management, and API bypass fees. It remains exceptionally lightweight and financially viable for a solo developer to maintain, launch, and scale.

## 🚀 Key Features

- **⚡ Blazing Fast:** Uses optimized DOM scanning to sort hundreds of items in milliseconds without freezing your browser.
- **🧮 "Read-the-Label" Logic:** Intelligently prioritizes Zepto's native discount labels (like "68% Off" on skincare) before falling back to raw MRP vs. Selling Price math.
- **🔒 100% Private & Safe:** Runs locally on your device. No API proxies, no external servers, and no session data is ever shared.
- **📱 Mobile Optimized:** Includes a specific scroll-loader for mobile that forces the app to load items instantly without manual scrolling.
- **🔍 Search Automation (PC):** Built-in sidebar to auto-search and sort popular categories with one click.
- **🛡️ Session State Preservation:** Automatically forces clicked items to open in a new tab (`noopener noreferrer`), preventing React's state-reload from wiping out your sorted list when you hit the back button.

## 🛠️ The Engineering Challenges & Solutions

Building a robust scraper without API access presented several unique edge cases:

1. **The ">50% Off" Bug & Savings Traps:**
   - _Challenge:_ Zepto cards display multiple numbers (e.g., MRP ₹100, Price ₹40, Save ₹60). Early logic picked the wrong numbers, resulting in incorrect calculations for deep discounts.
   - _Solution:_ Implemented strict "First-2 Numbers" array slicing and context-aware Regex to ignore non-price numbers (like EMI, months, and weights).
2. **The React Virtual DOM Text-Merge:**
   - _Challenge:_ High-speed `textContent` parsing merged adjacent HTML tags (e.g., `₹499` and `9% OFF` became `4999% OFF`), causing 9% items to sort to the top as 99%.
   - _Solution:_ Engineered the "HTML Spacing Trick" (`innerHTML.replace(/<[^>]+>/g, ' ')`) to preserve spaces between invisible tags without sacrificing the speed of native DOM reading.
3. **The Skincare "100% Off" Anomaly:**
   - _Challenge:_ Skincare items often hide native percentage labels without raw price differences, causing math fallbacks to divide backend paise by frontend rupees (resulting in fake 100% discounts).
   - _Solution:_ Switched to a "Label-First" extraction model that reads Zepto's native "X% OFF" UI badges before attempting fallback math.
4. **Bypassing WAF & CORS:**
   - _Challenge:_ Direct API calls to Zepto's backend result in 403 Forbidden errors due to strict Cloudflare/AWS WAF rules and CORS policies.
   - _Solution:_ Adopted the client-side Bookmarklet approach, operating as a "Trojan Horse" inside the user's authenticated, residential-IP browser session to completely nullify bot detection.
